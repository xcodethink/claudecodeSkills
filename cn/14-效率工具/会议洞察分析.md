# 会议洞察分析器（Meeting Insights Analyzer）

本技能将你的会议记录转化为关于你沟通模式的可操作洞察，帮助你成为更有效的沟通者和领导者。

## 适用场景

- 分析你在多次会议中的沟通模式
- 获取关于你的领导力和主持风格的反馈
- 识别你何时回避困难对话
- 了解你的说话习惯和填充词
- 追踪沟通技能随时间的改进
- 为绩效评估准备具体案例
- 辅导团队成员改善沟通风格

## 功能说明

### 1. 模式识别
识别会议中反复出现的行为，例如：
- 冲突回避或间接沟通
- 发言比例和轮流模式
- 提问 vs 陈述的比例
- 主动倾听指标
- 决策方法

### 2. 沟通分析
评估沟通效果：
- 清晰度和直接性
- 填充词和模糊语言的使用
- 语调和情感模式
- 会议控制和主持能力

### 3. 可操作反馈
提供具体的、带时间戳的示例：
- 发生了什么
- 为什么重要
- 如何改进

### 4. 趋势追踪
分析多次会议时比较随时间变化的模式

## 使用方法

### 基本设置

1. 将你的会议录音文字稿下载到一个文件夹（如 `~/meetings/`）
2. 在 Claude Code 中导航到该文件夹
3. 请求你想要的分析

### 快速开始示例

```
Analyze all meetings in this folder and tell me when I avoided conflict.
（分析此文件夹中的所有会议，告诉我什么时候我回避了冲突。）
```

```
Look at my meetings from the past month and identify my communication patterns.
（查看我过去一个月的会议，识别我的沟通模式。）
```

```
Compare my facilitation style between these two meeting folders.
（比较我在这两个会议文件夹中的主持风格。）
```

### 高级分析

```
Analyze all transcripts in this folder and:
1. Identify when I interrupted others
2. Calculate my speaking ratio
3. Find moments I avoided giving direct feedback
4. Track my use of filler words
5. Show examples of good active listening
```

## 详细指引

当用户请求会议分析时：

### 1. 发现可用数据
- 扫描文件夹查找文字稿文件（.txt、.md、.vtt、.srt、.docx）
- 检查文件是否包含说话人标签和时间戳
- 确认会议的日期范围
- 识别用户在文字稿中的名称/标识

### 2. 澄清分析目标

如果未指定，询问他们想了解什么：
- 具体行为（冲突回避、打断、填充词）
- 沟通效果（清晰度、直接性、倾听）
- 会议主持技能
- 发言模式和比例
- 需要改进的领域

### 3. 分析模式

对于每个请求的洞察：

**冲突回避**：
- 查找模糊语言（"maybe（也许）"、"kind of（有点）"、"I think（我觉得）"）
- 间接措辞而非直接请求
- 紧张时转移话题
- 没有承诺的同意（"yeah, but...（是的，但是...）"）
- 没有处理明显的问题

**发言比例**：
- 计算会议中发言的百分比
- 统计打断次数（用户打断他人和被打断）
- 测量平均发言轮次长度
- 追踪提问 vs 陈述的比例

**填充词**：
- 统计 "um"、"uh"、"like"、"you know"、"actually" 等
- 记录每分钟或每轮发言的频率
- 识别频率增加的情境（紧张、不确定）

**主动倾听**：
- 引用他人之前观点的问题
- 转述或总结他人的想法
- 在他人贡献上的拓展
- 提出澄清性问题

**领导力与主持**：
- 决策方法（指令式 vs 协作式）
- 如何处理分歧
- 是否包含较安静的参与者
- 时间管理和议程控制
- 跟进和行动项的清晰度

### 4. 提供具体示例

对于发现的每个模式，包含：

```markdown
### [模式名称]

**发现**：[模式的一句话总结]

**频率**：[在 Y 次会议中出现 X 次]

**示例**：

1. **[会议名称/日期]** - [时间戳]

   **发生了什么**：
   > [文字稿中的实际引用]

   **为什么重要**：
   [影响或错失机会的解释]

   **更好的方法**：
   [具体的替代措辞或行为]

[对 2-3 个最有力的示例重复]
```

### 5. 综合洞察

分析完所有模式后，提供：

```markdown
# 会议洞察摘要

**分析期间**：[日期范围]
**已分析会议**：[X 次会议]
**总时长**：[X 小时]

## 识别的关键模式

### 1. [主要模式]
- **观察到的**：[你看到了什么]
- **影响**：[为什么重要]
- **建议**：[如何改进]

### 2. [第二个模式]
[相同结构]

## 沟通优势

1. [优势 1 及示例]
2. [优势 2 及示例]
3. [优势 3 及示例]

## 成长机会

1. **[领域 1]**：[具体、可操作的建议]
2. **[领域 2]**：[具体、可操作的建议]
3. **[领域 3]**：[具体、可操作的建议]

## 发言统计

- 平均发言时间：[会议的 X%]
- 提问次数：[每次会议平均 X 个]
- 填充词：[每分钟 X 个]
- 打断：[每次会议给出 X 次 / 收到 Y 次]

## 下一步

[3-5 个改善沟通的具体行动]
```

### 6. 提供后续选项
- 在未来会议中追踪相同指标
- 深入分析特定会议或模式
- 与行业基准对比
- 创建个人沟通发展计划
- 为绩效评估生成摘要

## 示例

### 示例 1：冲突回避分析（灵感来源：Dan Shipper）

**用户**："我下载了所有会议录音并放在一个文件夹里。告诉我所有我暗中回避冲突的时刻。"

**输出**：
```markdown
# 冲突回避模式

在 15 次会议中发现 23 个你使用间接沟通或回避直面紧张局面的实例。

## 模式：在关键反馈上使用模糊表达

**频率**：在 7 次会议中出现 8 次

**示例 1：与 Sarah 的 1:1** - 00:14:32

**发生了什么**：
> "So, I was thinking... maybe we could, like, potentially
> consider looking at the timeline again? I mean, if you
> think that makes sense. But whatever you think is best!"
>（所以，我在想...也许我们可以，嗯，可能考虑再看看时间线？
> 我的意思是，如果你觉得有道理的话。但你觉得怎样好就怎样！）

**为什么重要**：
你需要指出 Sarah 的项目进度落后，但模糊的语言
（"maybe"、"potentially"、"if you think"）和最后的推卸
（"whatever you think"）使她很容易忽略紧迫性。

**更好的方法**：
"Sarah, the project is two weeks behind schedule. We need to
discuss what's blocking progress and create a new timeline
today."
（"Sarah，项目已经落后了两周。我们需要讨论是什么阻碍了进展，
今天就制定新的时间表。"）

[更多示例...]

## 建议

1. **在第一句话中直接说明问题**
2. **删除模糊词** 如 "maybe"、"kind of"、"sort of"
3. **提出具体问题** 而非暗示
4. **安排困难对话** 而非随意提起
```

### 示例 2：领导力主持分析

**用户**："分析我的团队会议，告诉我关于我的主持风格。"

**输出** —— 提供以下洞察：
- 你说话多少 vs 团队成员（60% vs 40%）
- 你是提问还是陈述（3:1 比例）
- 如何处理分歧（倾向于过快解决）
- 谁说话最少以及你是否引导他们参与
- 好的和错失的主持时刻的示例

### 示例 3：个人发展追踪

**用户**："比较我 Q1 和 Q2 的会议，看看我的倾听技能是否有改善。"

**输出** —— 创建对比分析：
- 打断次数减少（每次会议 8 次 → 3 次）
- 澄清性问题增加（每次会议 2 → 7 个）
- 在他人想法上拓展的改善
- 展示差异的具体示例
- 仍需成长的领域

## 设置建议

### 获取会议文字稿

**从 Granola**（Lenny's newsletter 订阅附赠免费版）：
- Granola 自动转录你的会议
- 将文字稿导出到文件夹
- 将 Claude Code 指向该文件夹

**从 Zoom**：
- 启用带转录的云录制
- 会议后下载 VTT 或 SRT 文件
- 存储在专用文件夹中

**从 Google Meet**：
- 使用 Google Docs 自动转录
- 将文字稿保存到文件夹
- 下载为 .txt 文件

**从 Fireflies.ai、Otter.ai 等**：
- 批量导出文字稿
- 存储在本地文件夹中
- 对该文件夹运行分析

### 最佳实践

1. **统一命名**：使用 `YYYY-MM-DD - Meeting Name.txt` 格式
2. **定期分析**：每月或每季度审查以发现趋势
3. **具体查询**：一次专注于一个行为以获得深度
4. **隐私保护**：敏感会议数据保存在本地
5. **行动导向**：一次聚焦一个改进领域
