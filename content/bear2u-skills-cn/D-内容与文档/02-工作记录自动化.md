# 工作记录自动化（Workthrough）

**AI 代理自动简要总结开发工作，并提出后续改进建议。**

## 核心要点

1. **自动执行**：开发工作完成时自动执行
2. **简要摘要**：AI 简洁整理工作内容
3. **包含建议**：提出待添加/改进的内容建议
4. **中文输出**：所有文档使用中文
5. **保存位置**：`<项目根目录>/workthrough/YYYY-MM-DD_HH_MM_description.md`

## 文档内容（保持简洁！）

```markdown
# [工作标题]

## 概述
用 2-3 句话总结做了什么

## 主要变更
- 开发内容：XXX 功能添加
- 修改内容：YYY Bug 修复
- 改进内容：ZZZ 性能优化

## 核心代码（仅在必要时）
```typescript
// 仅展示核心逻辑
const key = value
```

## 结果
- 构建成功
- 测试通过

## 下一步
- 接下来需要实现的功能
- 接下来需要修改的部分
```

## VitePress 处理

### 首次运行时（workthrough 文件夹不存在）
1. VitePress 初始设置
2. `pnpm install && pnpm dev` 测试

### 后续运行时（workthrough 文件夹已存在）
1. 仅创建新的工作记录文档
2. `pnpm build` 验证构建

## 重要提醒

- **简洁**：不要冗长说明
- **只留核心**：仅摘录重要代码
- **摘要**：说明开发/修改/改进了什么
- **下一步**：建议接下来应做的工作

完。
