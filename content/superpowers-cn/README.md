## 概述

这是一套完整的 **AI 驱动开发工作流技能体系**，从创意构思到代码上线形成闭环。每个技能都经过严格的 TDD 式测试验证，包含"铁律"、"反合理化表格"、"压力测试"等机制，确保 AI 在压力下仍能严格遵循最佳实践。

```
创意构思 → 方案设计 → 计划编写 → 任务执行 → 代码评审 → 分支完成
   ↑                                                        ↓
   └──────────── 验证 & 调试（贯穿全程）──────────────────────┘
```

## 体系核心设计模式

这套体系中最值得学习的 5 个设计模式：

1. **"铁律"模式** — 每个关键技能都有不可违背的核心规则，用大写加粗强调
2. **反合理化表格** — 预判 AI 可能找的借口并逐一封堵
3. **压力测试** — 用真实场景（生产事故、领导施压、赶工疲劳）验证技能在压力下是否有效
4. **双重评审** — 规范审查（是否按要求做）+ 质量审查（做得好不好），不同代理独立执行
5. **零上下文原则** — 计划写得任何人拿到都能直接执行，不依赖隐含上下文
